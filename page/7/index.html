java线程

发表于 2021-05-30 更新于 2021-08-09

新建线程并执行

法一：继承 Thread 类

public class myThread extends Thread {
    @override
    public void run() {
        try {
            // do something
        } catch(Exception e) {}
    }
    public static void main(String[] args) {
        new myThread().start();
    }
}

法二：实现 Runnable 方法

public class RunnableTest implements Runnable {
    @override
    public void run() {
        try {
            // do something
        } catch(Exception e) {}
    }
    public static void main(String[] args) {
        RunnbaleTest runnableTest = new RunnableTest();
        new Thread(runnbaleTest).start();
    }
}

实现接口的方式比继承类的方式更灵活，也能减少程序之间的耦合度

法三：直接创建 用于简单使用

Thread myThread = new Thread() {
    @override
    public void run() {
        try {
            Thread.sleep(7000);
        } catch(InterruptedException e) {
            e.printStackTrace();
        }
    }
};
myThread.start();

匿名类方式

new Thread(new Runnable() {
    @Override
    public void run() {
        try {
            // do sth.
        } catch(Exception e) {
            e.printStackTrace();
        }
    }
}).start();

start() & run()

  • 新建状态：使用 new 关键字和 Thread 类(或其子类)建立一个线程对象后，该线程对象就处于新建状态。它保持这个状态直到程序 start() 这个线程。
  • 就绪状态：当线程对象调用了start()方法之后，该线程就进入就绪状态。就绪状态的线程处于就绪队列中，要等待JVM里线程调度器的调度。
  • 运行状态：如果就绪状态的线程获取到 CPU 资源，就可以执行 **run()**，此时线程便处于运行状态。

只有调用了start()方法，才会表现出多线程的特性，不同线程的run()方法里面的代码交替执行。如果只是调用run()方法，那么代码还是同步执行的，必须等待一个线程的run()方法里面的代码全部执行完毕之后，另外一个线程才可以执行其run()方法里面的代码。

解决 Typora 编辑为知笔记频繁失焦问题

发表于 2021-05-10 更新于 2022-02-20

最近将 Typora 更新到了 0.10 版本，却发现编辑以前完美搭配的为知笔记内容时频繁失去窗口焦点，目测应该是新版 Typora 在自动保存后会切换焦点到同样进行保存行为的为知笔记上。

解决方法：在Typora for windows — dev version release下载，回滚历史版本（Windows），不用卸载，只需覆盖安装。

其他端版本亦如此点击 Change Log 进入历史发布版本界面。

2022 / 2 / 20 更新现状

Typora 在前不久结束了 Beta 测试，推出了正式版并进行买断式收费，89元的价格感觉挺合理而且可用支付宝付费，我推荐 [购买](Typora — a markdown editor, markdown reader.)。

并且更新后编辑为知笔记时的失焦问题也大大改善（有时还会出现），不过我认为也有可能是为知笔记的问题。

另外在前不久 为知笔记 也进行了新版的重构，而新版本中的实时渲染笔记是不兼容老版本的并且使用第三方编辑器编辑文本的功能被取消了，这令我选择停留在老版本。

附上保留的 typora 和 wiznote 的 安装包存档。

需求

PicGo设置GitHub图床

发表于 2021-03-31 更新于 2022-02-20

参考

  • .net core3.0上传文件出现404_PicGo+Github,实现Typora自动上传图片文件
  • PicGo+GitHub使用
  • 图床搭配 PicGo：打造高效的图片处理工作流

准备

  1. 注册一个 163 邮箱
  2. 用此邮箱注册一个专门存储图片的 GitHub 账号
  3. 生成仓库 用户名/仓库名
  4. 生成 token

全部勾选就好。

设置 PicGo

⚠ 注意：

  • 仓库名要写 用户名/仓库名 用户名是用来区分仓库的（用户名唯一）。
  • 分支名现在改成 main 了，不是原来的 master
  • 存储路径就是仓库中文件存储的路径，会自动生成文件夹。
  • 自定义域名用于图片加速 CDN 服务，一般不用填写，如果单纯想填就写 https://raw.githubusercontent.com/用户名/仓库名/main

如需 CDN 加速，jsdeliver 配置

  • 配置格式：https://cdn.jsdelivr.net/gh/用户名/仓库名
  • 举例：https://cdn.jsdelivr.net/gh/huayonglun/cdn_image001

就此就可以使用 GitHub 图床了。😸
