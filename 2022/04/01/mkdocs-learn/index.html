    Getting Started

    Material for MkDocs - Material for MkDocs (squidfunk.github.io)

    安装

    1. 使用 pip 安装：
    pip install mkdocs-material
    1. 使用 git 安装并使用 pip 安装所需环境：
    git clone https://github.com/squidfunk/mkdocs-material.git
pip install -e mkdocs-material

    建站

    初始化项目：

    mkdocs new .

    会产生如下目录结构：

    .
├─ docs/
│  └─ index.md
└─ mkdocs.yml

    基础配置

    配置目录下的 mkdoc.yml

    1. 使用 pip 安装的配置：
    theme:
  name: material
    1. 使用 git 安装的配置：
    theme:
  name: null
  custom_dir: mkdocs-material/material

  # 404 page
  static_templates:
    - 404.html

  # Necessary for search to work properly
  include_search_page: false
  search_index_only: true

  # Default values, taken from mkdocs_theme.yml
  language: en
  font:
    text: Roboto
    code: Roboto Mono
  favicon: assets/favicon.png
  icon:
    logo: logo

    发布网站

    ⭐MkDocs 支持对修改的内容的自动部署。

    GitHub Pages

    如果将项目托管在 GitHub 上，使用 GitHub Pages 是很方便的，可以使用 GitHub Actions 自动部署文档，或者通过命令行手动部署文档。

    使用 GitHub Actions

    • [ ] 学习
    name: ci 
on:
  push:
    branches:
      - master 
      - main
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-python@v2
        with:
          python-version: 3.x
      - run: pip install mkdocs-material 
      - run: mkdocs gh-deploy --force

    手动部署

    在包含 mkdocs.yml 文件的根目录使用以下命令行：

    mkdocs gh-deploy --force

    定制化

    添加资源

    MkDocs 提供了一些自定义主题的方法，例如添加 CSS 和 JavaScript 到 docs 目录。

    添加 CSS 和 JavaScript

    通过添加 CSS 来修改颜色或改变某些元素的间距。

    通过添加 JavaScript 来使用其他的高亮语法或添加一些逻辑操作。

    1. 在 docs 目录下新建 stylesheets 目录并放入所需的 CSS 文件，新建 javascripts 目录并放入所需 JavaScript 文件。

      得到以下目录结构：

    .
├─ docs/
│  └─ stylesheets/
│     └─ extra.css
│  └─ javascripts/
│     └─ extra.js
└─ mkdocs.yml
    1. 在配置文件 mkdocs.yml 中添加如下配置：
    extra_css:
  - stylesheets/extra.css
extra_javascript:
  - javascripts/extra.js

    扩展主题

    🟠 Material for MkDocs 是 MkDocs 的一个主题。

    如果要修改主题中 HTML 源码，就需要扩展主题。MkDocs 支持 主题插件，使用这种方法可以覆盖 Material for MkDocs 的一些组件而不需要从 git 拉取，这样在升级新版的时候就不会发生冲突问题。

    设置主题结构

    • 在 mkdocs.yml 中开启 Material for MkDocs 主题。
    • 创建一个 overrides 目录用来存放覆盖主题的文件，这个目录使用配置文件中的 custom_dir 配置。
    theme:
  name: material
  custom_dir: overrides

    覆盖目录的结构必须和原主题的结构保持一致，其目录下的任何文件都会替换原主题的同名文件。以后新增的资源也要放入此覆盖目录。

    原主题结构：

    .
├─ .icons/                             # Bundled icon sets
├─ assets/
│  ├─ images/                          # Images and icons
│  ├─ javascripts/                     # JavaScript files
│  └─ stylesheets/                     # Style sheets
├─ partials/
│  ├─ integrations/                    # Third-party integrations
│  │  ├─ analytics/                    # Analytics integrations
│  │  └─ analytics.html                # Analytics setup
│  ├─ languages/                       # Translation languages
│  ├─ content.html                     # Page content
│  ├─ copyright.html                   # Copyright and theme information
│  ├─ footer.html                      # Footer bar
│  ├─ header.html                      # Header bar
│  ├─ language.html                    # Translation setup
│  ├─ logo.html                        # Logo in header and sidebar
│  ├─ nav.html                         # Main navigation
│  ├─ nav-item.html                    # Main navigation item
│  ├─ palette.html                     # Color palette
│  ├─ search.html                      # Search interface
│  ├─ social.html                      # Social links
│  ├─ source.html                      # Repository information
│  ├─ source-file.html                 # Source file information
│  ├─ tabs.html                        # Tabs navigation
│  ├─ tabs-item.html                   # Tabs navigation item
│  ├─ toc.html                         # Table of contents
│  └─ toc-item.html                    # Table of contents item
├─ 404.html                            # 404 error page
├─ base.html                           # Base template
└─ main.html                           # Default page

    覆盖部分组件

    只需要在 overrides 目录下新建要替换的同名文件。目录结构如下：

    .
├─ overrides/
│  └─ partials/
│     └─ footer.html
└─ mkdocs.yml

    MkDocs 就会在渲染主题的时候使用新的组件。

    覆盖模板块

    除了覆盖部分组件，还可以覆盖或扩展组件模板来实现新的功能。（前者是整体替换，后者是部分修改）

    • 创建一个 main.html 文件在 overrides 目录：
    .
├─ overrides/
│  └─ main.html
└─ mkdocs.yml
    • 修改 main.html 来实现新的功能。

    详见 Customization - Material for MkDocs (squidfunk.github.io)

    主题开发

    Material for MkDocs 的构建基于 TypeScript、RxJS、SASS，因此如需更底层的更改就要修改主题的源码并且重新编译。

    详见 Customization - Material for MkDocs (squidfunk.github.io)

    Setup

    修改颜色

    Material for MkDocs supports Google’s original color palette, which can be easily configured through mkdocs.yml.

    可以在配置文件 mkdocs.yml 中设置一些基础颜色，也可以通过 CSS 来自定义颜色。

    配色方案

    Material for MkDocs 支持两种主题：

    • 明亮主题 default
    • 黑暗主题 slate

    在 mkdocs.yml 中进行如下设置：

    theme:
  palette:
    scheme: default

    基础颜色

    The primary color is used for the header, the sidebar, text links and several other components.

    基础颜色应用于页眉、侧边栏等多种组件。

    需要在 mkdocs.yml 中设置：

    theme:
  palette:
    primary: white

    颜色切换效果见 Changing the colors - Material for MkDocs (squidfunk.github.io)

    强调颜色

    The accent color is used to denote elements that can be interacted with, e.g. hovered links, buttons and scrollbars.

    强调颜色用来表示可交互的组件，例如链接、按钮、滚动栏等。

    在 mkdocs.ymml 中设置：

    theme:
  palette:
    accent: red

    颜色切换

    可以提供多种配色方案，用户可以自行切换。

    进行如下配置：

    theme:
  palette: 
    - scheme: default
      toggle:
        icon: material/toggle-switch-off-outline 
        name: Switch to dark mode
    - scheme: slate 
      toggle:
        icon: material/toggle-switch
        name: Switch to light mode

    图标的选择见 Changing the colors - Material for MkDocs (squidfunk.github.io)

    跟随系统配色

    跟随用户系统默认的外观选择配置方案，需要进行如下配置：

    theme:
  palette:
    - media: "(prefers-color-scheme: light)" 
      scheme: default
      toggle:
        icon: material/toggle-switch-off-outline
        name: Switch to dark mode
    - media: "(prefers-color-scheme: dark)" 
      scheme: slate
      toggle:
        icon: material/toggle-switch
        name: Switch to light mode

    修改字体

    配置

    Material for MkDocs 使用了 Google Fonts 更改字体很容易。另外也可以加载定制字体。

    Regular font

    在 mkdocs.yml 中常用字体可以很方便的设置成任意的 Google Font 字体集。

    theme:
  font:
    text: Roboto

    Monospaced font

    等宽字体用于代码块，可以单独设置样式，同样支持 Google Fonts。

    theme:
  font:
    code: Roboto Mono

    自动加载

    如果不想自动加载 Google Fonts 而是使用系统默认字体，只需要在 mkdocs.yml 中配置：

    theme:
  font: false

    自定义字体

    • [ ] 从另外的路径加载字体，需要使用额外的 CSS 来与 @font-face 交互。

    修改 logo 和图标

    配置

    Logo

    logo 可以更改为用户提供的 docs 目录下的图片或者与主题捆绑的图标。

    theme:
  logo: assets/logo.png
    theme:
  icon:
    logo: material/library

    通常来说，页眉和侧边的图标会链接到文档的首页即配置中的 site_url。但也可以自定义跳转的网址，通过以下配置：

    extra:
  homepage: https://example.com

    Favicon

    更改图标的配置：

    theme:
  favicon: images/favicon.png

    定制化

    自定义主题的图标：

    1. 新建一个 .icons 目录并放入所需的图标集，例如 Bootstrap。
      文件目录结构如下：
    .
├─ overrides/
│  └─ .icons/
│     └─ bootstrap/
│        └─ *.svg
└─ mkdocs.yml
    1. 在 mkdocs.yml 添加如下配置：
    markdown_extensions:
  - pymdownx.emoji:
      emoji_index: !!python/name:materialx.emoji.twemoji
      emoji_generator: !!python/name:materialx.emoji.to_svg
      options:
        custom_icons:
          - overrides/.icons

    设置导航栏

    配置

    即时装载

    theme:
  features:
    - navigation.instant

    How does the navigation work when using instant loading? · Discussion #2099 · squidfunk/mkdocs-material (github.com)

    本质是 XHR 技术，可以实现局部更新网页而不需要从新加载整个页面，能够提升加载效率。

    锚点跟踪

    theme:
  features:
    - navigation.tracking

    开启后，地址栏中末尾的锚点链接（#）会跟随着页面进入不同的小节而改变。

    开启导航栏

    theme:
  features:
    - navigation.tabs

    冻结导航栏

    theme:
  features:
    - navigation.tabs
    - navigation.tabs.sticky

    层级导航栏

    theme:
  features:
    - navigation.sections

    开启后左侧导航栏不再能够折叠。

    默认展开导航栏

    theme:
  features:
    - navigation.expand

    章节初始页

    theme:
  features:
    - navigation.indexes

    开启后会给每个章节添加一个新的初始页面，可以用来写一些前概大纲。

    需要在各自的目录新建 index.md 文件并添加到导航部分的最开头。

    nav:
  - Section:
    - section/index.md
    - Page 1: section/page-1.md
    ...
    - Page n: section/page-n.md

    两侧导航内容栏融合

    theme:
  features:
    - toc.integrate

    不好看，不建议使用。

    回到最顶按钮

    theme:
  features:
    - navigation.top

    使用标签

    在 mkdocs.yml 开启标签设置。

    plugins:

    添加标签

    When both, the built-in tags plugin and Metadata extension are enabled, tags can be added for a document with custom front matter.

    需要使用内置标签插件和 Metadata 插件。

    ---
tags:
  - insiders
  - brand new
---

    内置的标签插件允许用户可以创建一个标签初始页面 tags.md

    声明栏

    声明栏位于页面的最上方，用来展示项目新闻或重要消息。

    {% extends "base.html" %}

{% block announce %}
  <!-- Add announcement here, including arbitrary HTML -->
{% endblock %}

    插件

    Python Markdon

    Metadata

    支持在 Markdown 文件开头写上 yaml 语法的键值对。

    markdown_extensions:
  - meta

    Markdown in HTML

    通过使用这个插件，可以在 Markdown 中使用 HTML 的语法实现一些效果，如插入图片、卡片网格。

    markdown_extensions:
  - md_in_html

    Table of Contents

    设置右侧内容栏的标题和锚点的图标。

    markdown_extensions:
  - toc:
      permalink: true
      title: On this page

    更多 Markdown 的扩展语法见 Python Markdown Extensions - Material for MkDocs (squidfunk.github.io)

    SuperFences

    代码高亮

    markdown_extensions:
  - pymdownx.superfences

    多标签内容切换 Tabbed

    markdown_extensions:
  - pymdownx.tabbed:
      alternate_style: true

    详见 Python Markdown Extensions - Material for MkDocs (squidfunk.github.io)

    可以将关联的内容写在一起：

    === "A"

	sth..
	sth..
	
=== "B"

	sth ..
	sth ..

    Keys

    支持渲染按键组合的图标。

    markdown_extensions:
  - pymdownx.keys

    Highlight

    设置代码块高亮。此插件基于 SuperFences 来显示语法高亮，需要一起配置。

    markdown_extensions:
  - pymdownx.highlight:
      anchor_linenums: true
      use_pygments: true
      auto_title: true
      linenums: true
      linenums_style: pymdownx-inline
  - pymdownx.superfences

    use_pygments 是否使用 Pygments 来进行语法高亮，如果关闭则需要额外配置 docs/javascripts/highlight.js 和 mkdocs.yml：

    document$.subscribe(() => {
  hljs.highlightAll()
})
    extra_javascript:
  - https://cdnjs.cloudflare.com/ajax/libs/highlight.js/10.7.2/highlight.min.js
  - javascripts/highlight.js
extra_css:
  - https://cdnjs.cloudflare.com/ajax/libs/highlight.js/10.7.2/styles/default.min.css

    ❗其他的字段设置要设置使用 Pygments 来进行语法高亮。

    auto_title 自动显示代码的语言

    linenums 显示行号

    linenums_style 显示行号的风格 推荐使用 table or pymdownx-inline

    anchor_linenums 可以通过链接跳转行号 http://127.0.0.1:8000/mybatis/#__codelineno-7-1

    InlineHilite

    基于 Highlight，需要一起配置，可以为 inline code blocks 加上语法前缀高亮行内代码。

    markdown_extensions:
  - pymdownx.highlight
  - pymdownx.inlinehilite
  - pymdownx.superfences
