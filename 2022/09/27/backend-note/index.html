mybatis insert 返回自增主键

🟢方法一：

<-- xml 映射的方式 -->
<insert id="insert" parameterType="com.kebi.User" useGeneratedKeys="true" keyProperty="id" keyColumn="id">
	INSERT INTO user (
		username, password
	) VALUES (
		#{username}, #{password}
	)
</insert>
属性 描述
useGeneratedKeys 为 true 则返回主键的值
keyProperty 实体类中属性名
keyColumn 数据库主键字段名 如果和实体属性名一样可以不用写

执行插入数据操作之后，实体类数据会自动更新，但实体类中 id 属性字段一定需要 set 以及 get 方法。

@Service
public class UserService {
    @Autowired
    private UsertMapper userMapper;

    User user = new User();
    user.setUserName("name");
    user.setPassWord("123")

    userMapper.insert(user);

    System.out.println("获取自增主键:"+ user.getId());
}

insert、delete 的返回值是操作影响的行数，并不是返回的 id

🟢方法二：使用selectKey 查询主键

<-- xml 映射的方式 -->
<insert id="INSERT" parameterType="com.kebi.User">
    <selectKey resultType="int" order="AFTER" keyProperty="id" keyColumn="id">
        SELECT LAST_INSERT_ID()
    </selectKey>
    INSERT INTO users (
    	username, password
    ) VALUES (
    	#{username}, #{password}
    )
</insert>
属性 描述
resultType 结果类型
order 这可以被设置为 BEFORE 或 AFTER。如果设置为 BEFORE，那么它会首先选择主键，设置 keyProperty 然后执行插入语句。如果设置为 AFTER，那么先执行插入语句，然后是 selectKey 元素 - 这和像 Oracle 的数据库相似，在插入语句内部可能有嵌入索引调用.

Mybatis 自定义类型转换 TypeHandler

SpringBoot系列 Mybatis 之自定义类型转换 TypeHandler - SegmentFault 思否

SpringBoot系列 Mybatis 之自定义类型转换 TypeHandler - SegmentFault 思否 - Cubox

  1. mapper 中查询操作返回数据一般都是实体类的形式

  2. db 中的字段类型和 java bean 的属性类型最好对应，因为有默认的 typeHandler 处理类型转换 JdbcType.TIMESTAMP <-> TimeStamp，体现在 mapper 中的 @Results 注解。

    @Select("select * from money where id = #{id}")
// 指定 id 便于复用
@Results(id = "moneyResultMap", value = {
        @Result(property = "id", column = "id", id = true, jdbcType = JdbcType.INTEGER),
    	...,        
        @Result(property = "createAt", column = "create_at", jdbcType = JdbcType.TIMESTAMP),
        @Result(property = "updateAt", column = "update_at", jdbcType = JdbcType.TIMESTAMP, typeHandler = Timestamp2LongHandler.class)})
MoneyPo getById(@Param("id") int id);

  3. 但如果将 db 中的 timestamp 类型转换成 java bean 中的 Long 类型，就需要自定义 typeHandler。
    例子中使用的是 Timestamp2LongHandler ，继承了 BaseTypeHandler 类，泛型为 Long。
    实现两类方法：

    • setNonNullParameter：将 java 类型，转换为 jdbc 类型
    • getNullableResult：将 jdbc 类型转 java 类型

  4. 注册 typeHandler 的方法：

    • 🟢result 标签中显示指定，xml 或 注解

      <result column="update_at" property="updateAt" jdbcType="TIMESTAMP" typeHandler="com.git.hui.boot.mybatis.handler.Timestamp2LongHandler"/>

    • 【全局】在注入的配置类 MyBatisConfig 中指定

      @Configuration
public class MyBatisConfig {
  @Bean
  public SqlSessionFactory sqlSessionFactory() {
    ...
  }
}

    • 【全局】在 mybatis-config.xml 中注册，然后给 SpringBoot 指定此配置文件

      <configuration>
    <settings>
        <!-- 驼峰下划线格式支持 -->
        <setting name="mapUnderscoreToCamelCase" value="true"/>
    </settings>

    <typeHandlers>
        <typeHandler handler="com.git.hui.boot.mybatis.handler.Timestamp2LongHandler"/>
    </typeHandlers>
</configuration>

    • 【全局】直接在 SpringBoot 配置中指定 type-handlers-package

      mybatis:
  type-handlers-package: com.git.hui.boot.mybatis.handler

Java 8日期与数据库日期的映射关系

Java 8日期与数据库日期的映射关系 - 后端知识 - UJCMS

JDBC 4.2 规范

Java 日期 数据库日期
java.sql.Date DATE
java.sql.Time TIME
java.sql.Timestamp TIMESTAMP
java.util.Calendar TIMESTAMP
java.util.Date TIMESTAMP
java.time.LocalDate DATE
java.time.LocalTime TIME
java.time.LocalDateTime TIMESTAMP
java.time.OffsetTime TIME_WITH_TIMEZONE
java.time.OffsetDatetime TIMESTAMP_WITH_TIMEZONE

MySQL 数据库

  • date
  • dateTime，不受时区影响
  • time
  • timestamp 🔴 2038年溢出问题不能使用，读取会受 time zone 影响

MySQL甚至没有提供TIMESTAMP WITH TIME ZONE的类型，日期时间类型只有DateTime，没有 Time Zone 概念。必须使用jdbc连接中的serverTimezone确定时区。如jdbc:mysql://localhost/ujcms?serverTimezone=Asia/Shanghai。

给API和MySQL选择正确的时间类型

给API和MySQL选择正确的时间类型 - 简书 (jianshu.com)

对于数据库而言，时间处理的流程如：Java<--MyBatis(TypeHandler)转换-->JDBC<--转换->MySQL。数据库里面必须保存UTC时间。另外需要避免多余的转换，耗费性能。

  1. JDBC Timestamp 类型其实相当于Java DateTime类型，能包含年月日时分秒信息
  2. JDBC Timestamp 可以映射到MySQL的 timestamp 和 datetime。
    • 映射到 MySQL 的 timestamp 要考虑 Java 进程和 MySQL 的时区差异
    • 🟢映射到 MySQL 的 datetime （没有时区信息），JDBC 会根据 serverTimezone 做一次转换
  3. swagger-codegen会将type=string,format=date-time映射成OffsetDateTime
  4. SpringBoot对时间类型的属性提供了校验功能。比如对OffsetDateTime类型，加上@DateTimeFormat注解，就可以校验时间是否合法。
  5. Mybatis默认实现了很多TypeHandler用于做Java类型和JDBC类型之间的转换，比如OffsetDateTimeTypeHandler，用于处理OffsetDateTime到JDBC Timestamp类型的转换（无法自动处理 OffsetDateTime -> datetime，即 swagger 生成的是 OffsetDateTime，数据是 datetime）。

如何选择：

  • 🟢Java Bean 使用 datetime，数据库使用 datetime
    • 将Java服务进程的时区设置成UTC+0
    • 数据库URL设置serverTimezone=UTC，字段使用datetime类型，时间信息一目了然，使用SQL语句更加方便
  • Swagger 接口文档生成的 Java Bean 使用 OffsetDateTime，数据库使用 timestamp，JDBC 自动转换

🟠前端的角度

  1. 像后端传递 timestamp 最方便

  2. 传递 OffsetDateTime 需要进行转换

SpringMVC

概念

Spring MVC是Spring Framework的一部分，是基于Java实现MVC的轻量级Web框架。

MVC 框架

  • 模型 Model
  • 视图 View
  • 控制器 Controller

MVC主要作用是降低了视图与业务逻辑间的双向偶合。

最典型的MVC就是JSP + servlet + javabean的模式。

各部分的功能

Model：

  • 业务逻辑
  • 保存数据的状态

View：（JSP 或者其他视图模板 Thymeleaf / Freemarker）

  • 显示页面

Controller：（Servlet）

  • 取得表单数据
  • 调用业务逻辑
  • 转向指定的界面

==Spring的web框架围绕 DispatcherServlet（调度Servlet ）设计。==

controller配置

方法一：实现Controller接口，一个控制器中只有一个方法。

方法二：使用注解，同一个类中可以有多个方法，可以作为复用的模板。

@Controller
public class ControllerTest2 {
    @RequestMapping("/hello2")
    public String test2(Model model) {
        model.addAttribute("msg", "ControllerTest2");
        return "test"; // WEB-INF/jsp/test.jsp
    }

    @RequestMapping("/hello3")
    public String test3(Model model) {
        model.addAttribute("msg", "ControllerTest3");
        return "test"; // WEB-INF/jsp/test.jsp
    }
}

@RequestMapping 注解用于映射url到控制器类或一个特定的处理程序方法。可用于类或方法上。用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。

重定向和转发

设置 ModelAndView 对象，会根据 ViewName 和视图解析器跳转到：
{视图解析器前缀} + ViewName + {视图解析器后缀}

<!-- 视图解析器 -->
<bean class="org.springframework.web.servlet.view.InternalResourceViewResolver"
     id="internalResourceViewResolver">
   <!-- 前缀 -->
   <property name="prefix" value="/WEB-INF/jsp/" />
   <!-- 后缀 -->
   <property name="suffix" value=".jsp" />
</bean>

对应 Controller 类 🟠没用注解

RestFul风格

使用 Restful 操作资源，可以通过不同的请求方式来实现不同的效果。

http://localhost:8080/springmvc_04_controller_war_exploded/add/1/4

@Controller
public class RestFulController {
    @RequestMapping("/add/{a}/{b}")
    public String restful(@PathVariable int a, @PathVariable int b, Model model) {
        int result = a + b;
        model.addAttribute("msg", "结果是：" + result);
        return "test";
    }
}

MyBatis

mybatis – MyBatis 3 | Introduction

MyBatis 是一个 Java 持久层框架。

MyBatis 框架简化了之前学习 JDBC 时的繁琐步骤，更多的专注于配置文件，尤其是 mybatis-config.xml 和 mapper.xml ，并且也支持注解的形式来进行配置。

🟢配置 pom.xml 中依赖：

<!-- https://mvnrepository.com/artifact/org.mybatis/mybatis -->
<dependency>
  <groupId>org.mybatis</groupId>
  <artifactId>mybatis</artifactId>
  <version>3.5.7</version>
</dependency>
<!--  jdbc驱动包 -->
<dependency>
  <groupId>mysql</groupId>
  <artifactId>mysql-connector-java</artifactId>
  <version>8.0.11</version>
</dependency>

优点：

  1. 仅需要配置 sql 映射文件。
  2. sql 写在 xml 文件中，便于统一管理。
  3. 将业务逻辑和数据访问逻辑分离。
