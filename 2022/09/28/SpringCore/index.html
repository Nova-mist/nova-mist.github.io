Spring 核心基础

发表于 2022-09-28

基础概念

参考官方文档 -> Spring Framework

实际上我们在说 Spring、SpringMVC 的时候，其实是在说 Spring Framework，它由多个 Features 组成，例如：

  • 核心功能：AOP 面向切面编程，IOC 控制反转 / DI 依赖注入……
  • 测试模块
  • Data Access：JDBC 支持
  • Spring MVC 和 Spring WebFlux 网络框架

搭建项目

  1. 只需要导入下面的依赖就可以使用核心功能：
    • spring-webmvc
    • spring-jdbc
  2. Resources目录下的 springContext.xml 配置文件。
    将 java bean 注入到 Spring 容器中。

Core Technologies (spring.io)

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd">
    <!--bean就是java对象，由Spring创建和管理-->
    <bean id="hello" class="完整的包路径">
        <property name="name" value=""/>
    </bean>
</beans>

要实现不同的操作 , 只需要在 xml 配置文件中进行设置，即对象由Spring 来创建 , 管理 , 装配

依赖注入

基于 @Component + @Autowired 注解在类内使用

  • 属性注入
  • set 方法注入
  • 构造方法注入

基于配置文件

  • 构造注入
  • set 方法注入

基于 @Bean + @Autowired

  • 在 @Bean 作用的方法中手动创建（构造、setter）

详见 【Spring 概念】

  1. 属性注入
  2. setter方法注入
  3. 构造器注入

自动装配 autowire

Autowiring Collaborators

自动装配就是指 Spring 容器在不使用 <constructor-arg> 和<property> 标签的情况下，可以自动装配（autowire）相互协作的 Bean 之间的关联关系，将一个 Bean 注入其他 Bean 的 Property 中。

🟢不管有没有自动装配，都需要在beans.xml 配置文件中注册bean。（或使用 @Component 并配置注解扫描包）

Autowiring modes:

  • no
  • byName
  • byType 在容器中只有唯一的此类型对象
  • constructor 同byType但是constructor参数

Explicit dependencies in property and constructor-arg settings always override autowiring. You cannot autowire simple properties such as primitives, Strings, and Classes (and arrays of such simple properties). This limitation is by-design.

可以显示覆盖autowire；不能autowire primitive type，只适用于依赖的其他对象（可以不用写ref但java类中必须有setter）。

<bean id="Hello" class="com.ysama.pojo.Hello">
  <property name="name" value="spring"/>
  <!--        加了 autowire="byName" 可以省略-->
  <property name="bob" ref="bob"/>
</bean>
<bean id="bob" class="com.ysama.pojo.B">
  <property name="num" value="1"/>
</bean>

Using @Autowired

使用注解的自动装配甚至可以在java类中不写setter

需要在beans.xml 配置文件中开启注解支持：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           https://www.springframework.org/schema/beans/spring-beans.xsd
                           http://www.springframework.org/schema/context
                           https://www.springframework.org/schema/context/spring-context.xsd">

  <context:annotation-config/>

</beans>

@Autowired报红的时候检查是否导入了spring-webmvc依赖。

使用注解开发

💥就不用在 beans.xml 配置文件中注册beans了。

bean的实现

  1. 配置文档中引用context约束
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:context="http://www.springframework.org/schema/context"
      xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans.xsd
       http://www.springframework.org/schema/context
       http://www.springframework.org/schema/context/spring-context.xsd">

</beans>
  1. 配置扫描哪些包下的注解
<!--指定注解扫描包-->
<context:component-scan base-package="com.ysama.pojo"/>
  1. 在指定包下编写类，增加注解 默认类名的小写
@Component("user")
// 相当于配置文件中 <bean id="user" class="当前注解的类"/>
public class User {
   public String name = "";
}

属性注入

  1. 不适用 set 方法，直接在直接名上添加@value(“值”)

    @Component("user")
// 相当于配置文件中 <bean id="user" class="当前注解的类"/>
public class User {
   @Value("")
   // 相当于配置文件中 <property name="name" value=""/>
   public String name;
}

  2. 如果提供了set方法，在set方法上添加@value(“值”);

衍生注解 作用域

衍生注解：

  • @Controller：web层
  • @Service：service层
  • @Repository：dao层

作用域：

  • singleton 默认单例模式
  • prototype 多例模式
@Controller("user")
@Scope("prototype")
public class User {
   @Value("")
   public String name;
}
<context:component-scan base-package="com.ysama.pojo"/>
<context:annotation-config/>
  • 如果不扫描包，就需要手动配置bean。
  • 注册注解驱动，使注解生效。

通过Spring API实现

  1. 类实现API接口

public class Log implements MethodBeforeAdvice

public class AfterLog implements AfterReturningAdvice

  1. 在 Spring 文件中注册，实现aop切入实现，导入约束
<bean id="log" class="com.ysama.log.Log"/>
<bean id="afterLog" class="com.ysama.log.AfterLog"/>

<aop:config>
  <!--切入点 expression:表达式匹配要执行的方法-->
  <aop:pointcut id="pointcut" expression="execution(* com.ysama.service.UserServiceImpl.*(..))"/>
  <!--执行环绕; advice-ref执行方法 . pointcut-ref切入点-->
  <aop:advisor advice-ref="log" pointcut-ref="pointcut"/>
  <aop:advisor advice-ref="afterLog" pointcut-ref="pointcut"/>
</aop:config>
  1. 测试

自定义类实现 AOP

public class DiyPointcut {
   public void before(){
       System.out.println("---------方法执行前---------");
  }
   public void after(){
       System.out.println("---------方法执行后---------");
  }
}

在 Spring 中注册，定义了切面 aop:aspect

<bean id="diy" class="com.ysama.config.DiyPointcut"/>
<aop:config>
   <!--第二种方式：使用AOP的标签实现-->
   <aop:aspect ref="diy">
       <aop:pointcut id="diyPonitcut" expression="execution(* com.ysama.service.UserServiceImpl.*(..))"/>
       <aop:before pointcut-ref="diyPonitcut" method="before"/>
       <aop:after pointcut-ref="diyPonitcut" method="after"/>
   </aop:aspect>
</aop:config>

注解实现

@Aspect
public class AnnotationPointcut {
   @Before("execution(* com.ysama.service.UserServiceImpl.*(..))")
   public void before(){
       System.out.println("before method");
  }
   // @After
   // @Around
}
<!--第三种方式:注解实现-->
<bean id="annotationPointcut" class="com.ysama.config.AnnotationPointcut"/>
<aop:aspectj-autoproxy/>

🟠也可以自动注册 @Component 但要引入context约束 并且配置扫描哪些包下的注解
