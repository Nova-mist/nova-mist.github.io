5 初始化与清理

Constructor & GC

方法重载 P77

方法重载：每个重载的方法都必须有一个独一无二的参数类型列表。

  • 参数类型的不同
  • 参数顺序的不同（会使代码难以维护）
  • 不能通过返回值来区分

Java基本类型转换的优先顺序

byte, short, char < int < long < float < double

操作数1 | 操作数2 | 运算结果
byte short char | byte short char | int
byte short char | Int | Int
byte short char int | Long | Long
byte short char int long | Float | Float
byte short char int long double | Double | Double

如果传入重载方法的实际参数较大，要进行强制类型转换，否则报错。

如果已经定义了一个构造器（无论是否有参数），编译器不会自动创建默认构造器。

this P84

this 关键字在方法内部使用，表示对 “调用方法的那个对象” 的引用。

在构造器中调用构造器 P86

可以在构造器中使用 this 调用一个构造器，但不能使用两次。

除构造器之外，编译器禁止在其他任何方法中调用构造器。

终结处理和垃圾回收 P87

  • Java 有垃圾回收器负责回收无用对象占据的内存资源，这种机制使得没有析构函数
  • 垃圾回收器只知道释放经由 new 分配的内存，特殊情况需要自己写 finalize() 方法
  • 垃圾回收并不等于“析构”
  • Java 可以调用其他语言代码，带来内存泄漏，例如 没有 free() 的 malloc() （C语言）
  • Java 不允许创建局部对象，必须使用 new 创建对象，也没有用于释放对象的 delete

💡垃圾回收器如何工作 P89

”自适应的、分代的、停止—复制、标记—清扫“ 式垃圾回收器。

Java 中的垃圾回收器工作时，一面回收空间，一面使堆中的对象紧密排列，”堆指针“很容易移动到新分配的区域，避免了页面错误（频繁的内存调度，需要额外内存）。

  1. 引用记数是一种简单但速度很慢的垃圾回收技术，缺陷是对象之间的循环引用。

  2. 另一种思想：对任何活的对象，一定能最终追溯到其存活在堆栈或静态存储区之中的引用。

成员初始化 P92

  • 方法内的局部变量要初始化赋值。
  • 类内的每个基本类型数据成员会默认有初始值。对象引用默认为 null
  • 可以直接在定义类成员变量的地方为其赋值，C++中不可以

构造器初始化 P94

  • 自动初始化在构造器被调用之前发生
  • 变量定义顺序决定初始化顺序，但初始化先于其他方法与顺序无关
public class Counter {
    int i;
    Counter() {i = 7;}
    // i先为0然后变为7
}

创建顺序：静态初始化、非静态初始化、构造器、方法。

  • 静态初始化只有一次
  • 静态初始化中的非静态初始化、构造器都进行后才算完成一次静态初始化

显示的静态初始化 P97

static int i;
static {
    i = 47;
}

非静态实例初始化 P97

初始化每一个对象的非静态量，在构造器之前执行。

public class Mugs {
    Mug mug1;
    Mug mug2;
    {
        Mug1 = new Mug(1);
        Mug2 = new Mug(2);
    }
    Mugs() {}
    Mugs(int i) {}
}

数组初始化 P99

public class ArrayInit {
    public static void main(String[] args) {
        Integer[] a = {
            new Integer(1),
            new Integer(2),
            3, // autoboxing
        };
        Integer[] b = new Integer[] {
            new Integer(1),
            new Integer(2),
            3, // autoboxing
        };
        System.out.println(Arrays.toString(a));
        System.out.println(Arrays.toString(b));
    }
}

甚至可以在方法调用处创建数组，提供可替换的参数。

public class DynamicArray{
    public static void main(String[] args) {
        Other.main(new String[] {"i", "like", "happy", "life"});
    }
}
class Other {
    public static void main(String[] args) {
        for (String s : args) {
            System.out.print(s + " ");
        }
    }
}

可变参数列表 P102

static void f(int required, String... trailing) {
    System.out.print("required: " + required + " ");
    for (String s : trailing) {
        System.out.print(s + " ");
    }
}
public static void main(String[] args) {
    f(1, "one");
    f(2, "two", "three");
    f(0);
}

使用可变参数列表不依赖于 autoboxing ，使用的是基本类型。

但可以在单一的参数列表中将类型混合在一起，自动包装机制将有选择地将 int 参数提升为 Integer 

public static void f(Integer... args) {
    for (Integer i : args) {
        System.out.print(i + " ");
    }
}
public static void main(String[] args) {
    f(10, new Integer(11), 12);
}

只能在重载方法的一个版本上使用可变参数列表，或者都添加一个非可变参数。

枚举类型 P106

enum 关键字

命名都用大写字母表示（一个名字中有多个单词，使用下划线分隔开）

创建 enum 时，编译器会自动添加有用的特性：

  • 创建 toString() 方法
  • 创建 ordinal() 方法，用来表示某个特定 enum 常量的声明顺序
  • static values() 按照 enum 常量的声明顺序，产生由这些常量值构成的数组

经常搭配 switch 语句使用，并且可以使程序意图清晰。

public enum Spiciness {
    NOT, MILD, MEDIUM,, HOT, FLAMING
}
// switch 的新形式
switch (degree) {
            case NOT:
                System.out.println("not spicy at all");
                break;
            case MILD:
            case MEDIUM:
                System.out.println("a little hot");
                break;
            case HOT:
            case FLAMING:
            default:
                System.out.println("maye too hot.");
        }
//        switch (degree) {
//            case NOT -> System.out.println("not spicy at all");
//            case MILD, MEDIUM -> System.out.println("a little hot");
//            default -> System.out.println("maye too hot.");
//        }

// 声明的新写法
Burrito
                plain = new Burrito(Spiciness.NOT),
