学了点Unity以及小学期

发表于 2021-07-23

前些日子专业分了方向，选了媒体融合，应该可以不碰python了，但同时以后要接触很多Unity，所以小学期实践选了Unity。老师也是自话自说地讲课，内容竟然和之前看的书的内容很像，所以也没怎么听课，看了PlayMaker插件的视频、逐渐学会了搜谷歌翻官方文档。

使用Unity还是要多靠平时的积累，经验真的很重要，不然就要去查文档与google。

不过我真的不想深入学习Unity，因为我发现我自己并不适合做游戏（缺乏新奇的想法与思路）。我固然知道做游戏是有不同分工的（美术、策划、技术……），但我并不对与游戏相关的技术岗位感兴趣，其实连游戏都谈不上太感兴趣罢了。

我只不过是一个普通人，看着某同学行云流水般地制作游戏心里也很明确人与人之间的差距，还好我并不想走与游戏相关的道路，也就暗自为厉害的同学们加油吧。

Day02

搭建场景

  • 如果创建之前选中了某个物体，会创建子物体。
  • 在工程窗口新建材质文件

  • 新建 Texture 文件夹，指定材质球贴图
  • 材质拖拽到场景中或 Inspector 窗口中
  • 查看场景： 1. 预览窗口 2. “2 by 3” 窗口中游戏视图
  • 调整相机位置可以使用本地坐标系，方便调整 Z轴
  • 添加球体 Sphere 并在 Inspector 窗口中添加 rigid 刚体组件

正交3D游戏

  • 3D模型
  • 正交摄像机代替透视摄像机，没有近大远小的效果，适用于鸟瞰视角的游戏，如摩尔庄园手游。

组件

物体只是装载组件的容器，组件是实现具体游戏功能的基石。所有的游戏物体默认至少具有变换组件，定义位置和方向信息。

组件选项菜单：

  • Reset
  • Remove Component 某些组件之间具有依赖关系
  • Move Up/Down
  • Copy/Paste Component

游戏运行时修改的组件参数不会保存

脚本与组件操作

  • 编写脚本，并作为组件挂载到游戏物体上。
  • 每一个新的脚本都定义了一种新的组件类型。
  • 每个类都继承 MonoBehaviour
  • 挂载脚本到物体上实际上是创建了该脚本的实例。
  • 通常不使用构造函数
  • update() 函数会在每一帧调用，用来实现用户输入、查找、修改该组件信息等。

访问组件

void Start() {
    Rigidbody rb = GetComponent<Rigidbody>();
    // 改变刚体的质量为10kg
    rb.mass = 10f;
    // 施加向上的力
    rb.AddForce(Vector3.up * 10f);
}
  • 用变量引用游戏物体，被拖拽的游戏物体只要有 public Transform playerTransform 组件就可以

查找子物体

public Transform[] waypoints;
    // Start is called before the first frame update
    void Start()
    {
        waypoints = new Transform[transform.childCount];
        int i = 0;
        foreach(Transform t in transform)
        {
            waypoints[i++] = t;
        }
        transform.Find("Gun");

常用事件函数

传统的游戏：在一个主循环中处理游戏逻辑。

Unity：在特定的事件发生时，调用脚本中特定的函数。

void Update() {
    float distance = speed * Time.deltaTime * Input.GetAxis("Horizontal");
    transform.Translate(Vector3.right * distance);
}

物理引擎的更新在 FixedUpdate() 函数中

void FixedUpdate() {
    Vector 3 force = transform.forward * driveForce * Input.GetAxis("Vertical");
    rigidbody.AddForce(force);
}

更新摄像机的位置

void LateUpdate() {
    Camera.main.transform.LookAt(target.transform);
}

解决帧率变速

将两帧之间的事件间隔 Time.deltaTime 考虑进去，会随游戏环境改变。引擎默认60fps

transform.Translate(0, 0, distancePerSecond * Time.deltaTime);

TimeManager 修改物理更新的时间间隔。对应脚本中 Time.fixedDeltaTime

创建与销毁

public GameObject enemy;
void Start() {
    for (int i = 0; i < 5; i++) {
        Instantiate(enemy);
    }
}
// 子弹碰撞时 0.5s后销毁
void OnCollisionEnter(Collision otherObj) {
    if (otherObj.gameObject.tag == "Missile") {
        Destroy(gameObject, 0.5f);
    }
}

注意：Destroy(this) 会从物体上删除脚本组件而不是销毁物体。4

标签

层级 Layer

用来让摄像机仅渲染场景中的一部分物体，让灯光只照亮一部分物体，碰撞检测、射线检测。

配合摄像机的 Culling Mask ，选择性显示某些层级的物体。

选择性的射线检测

int 型变量的32个位来表示，0不发生碰撞

int layerMask = 1 << 8;
// 和非自身物体碰撞
// layerMask = ~layerMask;
RaycastHit hit;
if (Physics.RayCast(transform.position, Vector3.forward,
                   Mathf.Infinity, layerMask));

预制体

区别于模型的引用。预制体是在场景中创建物体的模板，既可以关联修改所有物体，也可以由单个物体重载一些组件和参数。

在运行时实例化预制体。

建造一堵墙

  • 创建空白物体并挂载脚本
  • 创建具有刚体组件的预制体并关联到脚本
public Transform brick;
    // Start is called before the first frame update
    void Start()
    {
        for (int y = 0; y < 5; y++)
        {
            for(int x = 0; x < 5; x++)
            {
                Instantiate(brick, new Vector3(x, y, 0), Quaternion.identity);
            }
        }
    }

多个物体按特定的形状摆放

public GameObject prefab; // 预制体
public int numberOfObjects = 20; // 物体总数
public float radius = 5f; // 圆圈半径
void Start() {
    for (int i=0; i < numberOfObjects; i++) {
        // 算出物体间隔角度
        float angle = i * Mathf.PI * 2 / numberOfObjects;
        // 利用三角函数求位置
        Vector3 pos = new Vector3(Mathf.Cos(angle), 0, Mathf.Sin(angle)) * radius;
        Instantiate(prefab, pos, Quaternion.identity);
    }
}
// 矩形
public GameObject prefab;
public float gridX = 5f;
public float gridY = 5f;
public float spacing = 2f; // 物体之间间隔
void Start() {
    for (int y=0; y < gridY; y++) {
        for (int x=0; x <gridX; x++) {
            Vector3 pos = new Vector3(x, 0, y) * spacing;
            Instantiate(prefab, pos, Quaternion.identity);
        }
    }
}

移动设备的输入

Input.touches 多点触摸

实例：用户点击屏幕发射一条射线，射线碰触到物体以后，实例化一个预制体。

  • 脚本挂载到 Main Camera 上面
  • 在（0，0，0）点放置一个平面，让射线碰撞
  • 创建一个预制体，并关联到脚本上
  • 运行游戏，单机地面进行实例化
public GameObject prefab;
// Update is called once per frame
void Update()
{
    //foreach(var touch in Input.touches)
    //{
    // 某个手指刚开始接触
    //if(touch.phase == TouchPhase.Began)
    if (Input.GetMouseButtonDown(0))
    {
        // 利用摄像机和屏幕上的点 确定出一条从手指到场景内的射线
        //var ray = Camera.main.ScreenPointToRay(touch.position);
        var ray = Camera.main.ScreenPointToRay(Input.mousePosition);
        // 用物理引擎发射射线，碰到物体返回 true
        RaycastHit hitInfo;
        if (Physics.Raycast(ray, out hitInfo))
        {
            // 如果碰到 碰撞信息保存在 hitInfo
            Instantiate(prefab, hitInfo.point, Quaternion.identity);
        }
    }
    //}
}

在移动平台的游戏的开发阶段暂时用鼠标操作代替触屏操作。

方向与旋转

  • 欧拉角，简单直观，但会遇到万向节锁定问题（Gimbal Lock）
  • Unity 的旋转工具使用四元数系统，不会产生万向节锁定，但一个四元数无法超过 180°旋转，数值不直观。
  • 检视窗口中还是以欧拉角表示旋转，引擎内部是四元数，所以超过角度的数值会自动改变。
transform.Rotate(0,30,0); // 绕Y轴转30°

灯光

  • 灯光组件可以直接添加到游戏物体上，Rendering 分类。
  • 不同的渲染路径影响光照和阴影。

灯光分类：

  • 点光源，空间中的一个点向每个方向都发射同样强度的光。光照强度与距离的平方成反比。模拟灯泡、蜡烛、枪口的照明。
  • 探照灯，发射角度被限制在一个固定的角度，人造光源。
  • 方向光，自然光，没有发射源位置，即所在的位置不会对效果产生影响。修改 Skybox 的设置。
  • 矩形光，光线柔和
  • 发光材质，shader 中的 Emission 勾选
  • 环境光，不属于组件， 在 Scene > Environment Lighting

摄像机

作为组件挂载到物体上。

渲染贴图

Render Texture 将摄像机拍摄到的画面渲染到一张贴图上，镜子、显示器、小屏幕、监视器、画中画。

视图矩阵

Normalized Viewport Rectangle 用来指定摄像机所拍摄的内容固定显示在屏幕的某一个矩形范围内。小地图、双人分屏。

剔除遮罩

Culling Mask

仅深度

用于混合两个摄像机的画面使其不叠加。

摄像机根据深度先后绘制，后一个保留前一个的画面但清除深度信息，画面会叠加。

应用：FPS游戏中，枪放在远离场景的位置用另一个摄像机单独渲染再叠加到游戏画面，不会被物体挡住。

Day04

资源工作流程

资源导入

在文件浏览器中移动或重命名文件，要同样修改 .meta 文件。所以只在 Unity 编辑器下操作。

导入3D建模软件的自有3D格式的文件，Unity 会回调3D软件的FBX导出插件。

物理

3D引擎和2D引擎。

  • 刚体，不建议通过在脚本中直接修改 Transform 属性而是通过对刚体施加力来推动物体。
  • 休眠 Sleeping
  • 碰撞体 Collider，通常使用近似的物理形状，组合碰撞体
    • 静态碰撞体，地形、障碍物
    • 刚体碰撞体
    • Kinermatic 让物体在物理系统控制和动画系统控制之间切换
  • 物理材质 3D Physic Materials / 2D Physics Material 2D
  • 触发器，Is Trigger ，OnCollisionEnter 方法
  • 碰撞事件触发表，Collision Message / Trigger Message
  • 物理关节 Joints
  • 角色控制器，因为刚体组件会让人物难以控制。

在Unity 中使用人物模型，要确保人物的高度在2米左右，两个默认 Cube

阻尼：0.001（金属）~ 10（羽毛）

常量力组件

  • Force 世界坐标力
  • Relative Force 局部坐标力
  • Torque 世界坐标扭矩 越大转越快
  • Relative Torque

Day05

动画

  • 模型动画：定点动画、骨骼动画
  • 非模型动画：对游戏内游戏物体的动画，通过曲线编辑工具配置

动画混合的核心——插值与权重

在 Unity 引擎中，任何功能系统最后都可以简单拆分为三个模块：资源模块、控制&编辑模块、实体模块。

动画系统工作流程

动画剪辑：资源模块，对象如何随着事件改变属性信息。

控制器：记录动画之间的关系，用来对动画剪辑进行管理。Animator Controller Window > Animator 流程的窗口

动画组件：Animator 游戏实际运行时动画功能的实体模块，对当前对象读取动画控制器、设置动画刷新时间。所有动画剪辑都存放于动画控制器中。

动画剪辑

  • Model
  • Rig
  • Animation
  • Materials

人形角色动画

Avatar：一个骨架结构到用户实际骨架结构的映射。

  • 在导入模型资源的 Rig 选项中 修改 Animation Type 参数为 Huamnoid
  • 将非关键骨骼也进行匹配，并使模型处于T形姿态
  • 骨骼映射信息、存储信息保存为一个人形模板文件 xx.hit 可以复用

人形动画身体遮罩

控制身体的一部分受动画影响，其他部分被屏蔽。

人形动画的重定向

重新利用现有的动画以供多个模型使用，从而无须创建全新的动画方法。

只支持人形模型（可以创建骨架映射）。

两种形式：

  • 两个模型使用相同的骨架，所有层级和命名都相同，因为动画根据名字和路径去查找操作的骨骼。
  • 通过 Avatar ，让动画通过映射在多个不同骨架下产生相同的模型动作。
    • 拖入动画模型，在模型根节点上创建一个动画控制器并加入一组动画片段。
    • 在挂载的动画控制器的 Avatar 选项中生成人形骨架映射 Avatra 。
    • 拖入另一个模型，同样创建动画控制组件。
    • Animator 下将 Avatar 选项拖入自身的 Avatar ，Controller 选项选择之前模型的动画控制器。

  • 正向运动 FK ：通过各个关节的扭动实现
  • 逆向运动 IK：由末端结点控制整体位置

默认脚用 IK，手臂用 FK。如果手要拿东西或贴在物体上切换 IK。
