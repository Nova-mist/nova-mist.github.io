关于windows上的软件配置文件

发表于 2021-07-27

看了一篇文章一日一技 | 集中管理与备份 Windows 分散各处的应用配置文件 - 少数派 (sspai.com)，记录一些关于软件配置文件的知识。🎉

配置文件的所在

Windows 是一个多用户的操作系统，从注册表，到文件目录都严格遵循「系统 - 用户」的两级排列方式，配置文件也不例外。

  1. 系统盘根目录隐藏文件夹：ProgramData。存放以管理员权限安装的各类应用的默认设置。
  2. 用户目录隐藏文件夹：AppData\Roaming。存放适用于此用户的应用配置。
  3. 命令行工具，如 Vim，Git，Nodejs 的包，其配置文件一般在用户目录。
  4. 绿色软件配置文件在自己的文件夹内，UWP 软件被严格隔离在沙盒内。

locallow：共享数据存放文件，一般都可以清理一些无用的共享文件。

Local：本地保存文件，其中本地临时文件，AppData\Local\Temp\下面的文件可以删除（注意是Temp）。

Roaming：保存应用程序运行后的数据信息，如果删除应用程序运行配置数据会丢失。

链接文件/文件夹

遇到文件就使用硬链接，遇到目录就使用目录链接。

工具 ：Link Shell Extension。

  1. 创建一个同步文件夹。
  2. 将配置文件/文件夹移动到同步文件夹。
  3. 创建链接。

同步文件夹中可以放真正的文件，文件更不易丢失，但如果出错就放入链接文件。

使用Git备份

  1. 使用 .gitignore 忽略掉无需跟踪的二进制文件。

  2. .gitignore 创建在 Git 库的根目录时对所有文件生效，创建在子文件夹时仅对该文件夹生效。

  3. 提交到远程仓库。

    git add .
git commit -m "Settings 2021-07-21"
git remote add origin https://github.com/example/example.git
git push -u origin master

  4. 设置定时任务。

    $date = Get-Date -Format 'yyyy-MM-dd HH:mm'
cd $home/Settings
git add .
git commit -m "Autobackup $date"
git push -q # quiet 安静模式

    在「任务计划程序」中设置自动执行。
