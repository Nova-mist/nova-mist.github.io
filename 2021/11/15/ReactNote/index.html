React哲学

准备：在编写前段代码之前需要：

  1. UI 设计稿
  2. 后端 API 返回的 JSON 数据

第一步：将设计好的 UI 划分为组件层级

  • 一个组件原则上只能负责一个功能
  • 每个组件需要与数据模型的某部分匹配
  • 列出一个详细名称的层级表示

第二步：用 React 创建一个静态版本

  • 用已有的数据模型渲染一个不包含交互功能的 UI。
  • 通过 props 传入所需的数据，不应该使用 state 构建静态版本。

你可以自上而下或者自下而上构建应用：自上而下意味着首先编写层级较高的组件（比如 FilterableProductTable），自下而上意味着从最基本的组件开始编写（比如 ProductRow）。当你的应用比较简单时，使用自上而下的方式更方便；对于较为大型的项目来说，自下而上地构建，并同时为低层组件编写测试是更加简单的方式。

第三步：确定 UI state 的最小（且完整）表示

只保留应用所需的可变 state 的最小集合，其他数据均由它们计算产生。

属于 state 的数据：

  1. 随时间发生改变的数据
  2. 不能由其他数据计算出
  3. 不是由父组件通过 props 传递而来

第四步：确定 state 放置的位置

将低层级使用的 state 存放到共有的高层级中，通过 props 向下传递数据。

第五步：添加反向数据流

  1. state 只能由拥有它们的组件进行更改。
  2. 高层组件将回调函数（callback）传递给低层组件调用。通常绑定响应事件的函数 onChange 等，借助高层组件的 handle 方法调用 setState() 更新应用。
