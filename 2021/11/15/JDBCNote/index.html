4 数据库权限与设计

4.1 用户管理

狂神说MySQL07：权限及如何设计数据库 (qq.com)

使用 SQLyog 创建用户，赋予权限。

root 用户拥有最高的权限，可以赋予其他用户除了 GRANT OPTION 之外的所有简单权限。

用户的信息存放在 mysql.user 表中

-- 刷新权限
FLUSH PRIVILEGES
-- 增加用户
CREATE USER `[username]` IDENTIFIED BY `[password]` 
-- 重命名用户
RENAME USER old_user TO new_user
-- 设置密码
SET PASSWORD = PASSWORD('密码')    -- 为当前用户设置密码
SET PASSWORD FOR [user] = PASSWORD('密码')    -- 为指定用户设置密码
-- 删除用户
DROP USER [username]
-- 分配权限/添加用户
GRANT 权限列表 ON 表名 TO 用户名 [IDENTIFIED BY [PASSWORD] 'password']
  - all privileges 表示所有权限
  - *.* 表示所有库的所有表
  - 库名.表名 表示某库下面的某表
-- 查看权限
SHOW GRANTS FOR [username]
SHOW GRANTS FOR CURRENT_USER; -- 当前用户
-- 撤销权限
REVOKE 权限列表 ON 表名 FROM 用户名
REVOKE ALL PRIVILEGES, GRANT OPTION FROM 用户名    -- 撤销所有权限

4.3 三大范式

数据库设计三大范式 - Ruthless - 博客园 (cnblogs.com)

  1. 第一范式：所有字段值都是不可分解的原子值（地址的拆分）
  2. 第二范式：确保表中的每列都和主键相关（联合主键的订单使用3个表）
  3. 第三范式：每列都和主键列直接相关,而不是间接相关（订单表中只有客户编号，不可有其他客户信息）

5.7 JDBC操作事务

  1. 开启事务。
  2. 一组事务执行完毕，提交事务。
  3. 在catch语句中显示定义回滚语句，默认失败回滚。
Connection conn = null;PreparedStatement st = null;ResultSet rs = null;try {    conn = JdbcUtils.getConnection();    // 关闭自动提交 开启了事务    conn.setAutoCommit(false);    String sql1 = "UPDATE account SET cash=cash-100 WHERE name='A'";    st = conn.prepareStatement(sql1);    st.executeUpdate();    // int x = 1/0;    String sql2 = "UPDATE account SET cash=cash+100 WHERE name='B'";    st = conn.prepareStatement(sql2);    st.executeUpdate();    // 事务完毕 提交    conn.commit();    System.out.println("成功");} catch (SQLException throwables) {    try {        // 失败回滚        conn.rollback();    } catch (SQLException e) {        e.printStackTrace();    }    throwables.printStackTrace();} finally {    JdbcUtils.release(conn,st,rs);}
