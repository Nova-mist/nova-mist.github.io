基础概念

实际上我们在说 Spring、SpringMVC 的时候，其实是在说 Spring Framework，它由多个 Features 组成，例如：

  • 核心功能：AOP 面向切面编程，IOC 控制反转 / DI 依赖注入……

搭建项目

    • spring-webmvc

  2. Resources目录下的 springContext.xml 配置文件。
    将 java bean 注入到 Spring 容器中。

Core Technologies (spring.io)

ssm-jsp 配置环境

发表于 2022-09-27

SpringMVC + JSP 使用视图解析器将 /hello 请求映射到 /WEB-INF/jsp/hello.jsp

  1. 创建项目，【Add FrameWork Support】添加 WebApp 依赖。

  2. 在 pom.xml 引入 spring-webmvc 依赖。（目前不需要 servlet jsp 相关依赖）

  3. 在 web 目录添加相应文件，配置 web.xml 文件

    • 设置 <servlet> 和 <servlet-mapping>
    • 关联 SpringMVC 配置文件

  4. 配置 spring-mvc.xml

    • 开启注解支持(默认开启)
    • 设置自动扫描包
    • 设置不处理静态资源
    • 开启支持 mvc 注解驱动
    • 设置视图解析器拼接前缀、后缀

    添加处理器映射器。

    添加处理器适配器。

    添加视图解析器。Spring 默认的，也可指定 ThymeLeaf / Freemarker 等

    使用springMVC必须配置的三大件：

    处理器映射器、处理器适配器、视图解析器

    通常，我们只需要手动配置视图解析器，而处理器映射器和处理器适配器只需要开启注解驱动即可，而省去了大段的xml配置

  5. 编写一个 Controller。

    • 使用 @Controller 注入 bean
    • @RequestMapping 映射请求路径

  6. 【Ctrl + Alt + Shift + S】打开 Project Structure， 选择设置 Artifacts，在 /WEB-INF 目录下新建 lib 目录并添加 SpringMVC 相关依赖。

  7. 配置 Tomcat 并启动，需要将 Deployment 菜单的 Application Context 设为 /

注意：如果运行出现问题，要检查 out 目录下的 war 包结构，WEB-INF 目录下的 classes 和 lib 是平级的。

Docker 学习笔记

DevOps

基于 Go 开发

Reference

Home - Docker

Docker Hub

什么是docker，它与虚拟机有什么区别？_itmkyuan的博客-CSDN博客_docker和虚拟机的区别

概念

Docker 解决依赖的兼容问题：

  1. 将应用的函数库、依赖、配置与应用一起打包。
  2. 将每个应用放到一个容器去运行，避免互相干扰。

系统环境特点：

  1. Linux系统的内核都是相同的。
  2. 操作系统提供的函数库是对内核指令的封装。

Docker 与虚拟机的区别：

  • 虚拟机是在操作系统中模拟硬件设备，再运行另一个操作系统。
  • Docker只封装了库函数，直接使用主机的操作系统内核。

DevOps 开发运维

直接对镜像进行操作。

容器化技术使开发和测试的环境高度一致。

Docker 是内核级别的虚拟化，有着更高效的计算资源利用。

Docker 的基本组成：

  • 镜像 Image
  • 容器 Container
  • 仓库 Repository

底层原理

Docker 是一个 Client - Server 结构的系统。Docker 的守护进程运行在主机，客户端通过 socket 进行访问。

把Line表情转换成微信表情

发表于 2022-07-13

把Line表情转换成微信表情

准备工作

  • 安装Chrome插件 line贴图打包
  • 在 line商店 找到想要的表情包并下载
  • 安装 image magick

转换格式

🟢对于静态图片，只需要使用Image magick的命令行。

# 新建文件夹
mkdir .\output
# 获取路径下所有表情文件
$files = Get-ChildItem .\
# 转换每一个表情
foreach ($file in $files) {magick "$file" "$file".Replace("png","gif")}
# 将所有转换好的gif图片移动到 output文件夹
Get-Item *.gif | Move-Item -Destination .\output

名字带有 key 的是小图，正常微信表情适合用大图。

🟠对于动态图片，因为是apng的格式，一般的转换传到微信会显示错误，只能使用一个日语的转换网站 Magical Converter 来转成gif。

  • 一般动图的fps设定为 0.02秒（50fps），每帧的播放时间。本着APNG的总播放时间不能超过4秒参考，自行调整。
  • 对于还是会出现闪出白色背景的动图
    • 可以将背景颜色设定为#ededed，这样在手机上不会看出闪动，但电脑上会显示闪几帧
    • 也可以取消背景透明的选项，再改背景颜色，这样在电脑上看背景是固定的颜色

测试的图片：

  • 静态：Office Worker ButterCat – LINE貼圖 | LINE STORE
  • 动态：星のカービィ まんまるスタンプ - LINE スタンプ | LINE STORE
